# Step 2 — MSK 클러스터 (KAN-66)

## 생성 설정 (기록)
| 항목 | 값 |
|---|---|
| 클러스터 이름 | `hf4-msk` |
| 유형 | 프로비저닝됨 |
| Kafka 버전 | 3.9.x (KRaft) |
| 브로커 | `kafka.m7g.large` × 2 (t3.small 미지원 리전) |
| 스토리지 | 1 GiB/브로커 |
| VPC | `vpc-05586f35a27a6761a` (hf4-vpc) |
| 서브넷 | `subnet-088ccaf9f42333141` (2a), `subnet-098974f805aa26f1b` (2b) |
| 보안 그룹 | `sg-075ff3b098bffa7e9` (default) |
| 인증 | **IAM 역할 기반** (TLS 필수, 일반텍스트 불가) |
| AWS 계정 | `601202752151` |
| 리전 | ap-northeast-2 |

## 생성 후 할 일
1. 상태 **활성** 되면 → **[클라이언트 정보 보기]** 에서 **부트스트랩 서버(IAM용, 포트 9098)** 주소 복사 → Step 3에 사용
2. **퍼블릭 액세스** 필요 시(노트북에서 발행): 클러스터 → 편집 → 퍼블릭 액세스 켜기 (~30분). 안 켜면 VPC 내부(EC2)에서만 접근.

## 💸 삭제 (테스트 종료 후 필수)
```
MSK Connect 커넥터 삭제  →  MSK 클러스터 삭제
```
상시 방치 시 월 ~$160+. 켜둔 시간만 과금.
